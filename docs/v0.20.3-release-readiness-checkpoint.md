# GURPS QuickDeck v0.20.3 Release-readiness checkpoint

## 1. Executive summary

This v0.20.3 checkpoint is a conservative release-readiness review after UI1 retirement, UI2-only cleanup, drag/drop hardening, Forge/path hardening, Foundry API lifecycle auditing, and runtime QA diagnostics.

Findings:

- QuickDeck is ready to move from the v0.20 hardening line into UI2 polish.
- README and CHANGELOG now describe v0.20.3 as a UI2-only release-readiness checkpoint.
- `module.json` is aligned to version `0.20.3` and its declared script/style/readme/license paths exist.
- No active runtime reference to deleted UI1 templates was found in `scripts/`, `templates/`, `styles/`, `module.json`, `README.md`, or the current top-level `CHANGELOG.md` release wording.
- Remaining `.quickdeck-*` references are shared/core/runtime selectors or historical documentation, not a supported UI1 interface.
- All CSS `url(...)` references in `styles/*.css` resolve to existing module files.
- No active runtime hardcoded local install paths were found for `file://`, `/Data/`, `AppData`, or `C:\`.
- No JS behavior, HBS behavior, CSS layout/visuals, roll/action/roster/pending-damage behavior, `data-action` names, dependencies, or binary assets were changed.

## 2. UI2-only status confirmation

QuickDeck is UI2-only for this checkpoint. The supported user interface is the `qd40` / `qd-ui2` overlay rendered from `templates/quickdeck-overlay.hbs` through the current Foundry host lifecycle. The retired UI1 app shell and sidecar templates are not supported release surfaces.

Review notes:

- `README.md` states QuickDeck remains UI2-only and that retired UI1 app shell/sidecar templates/old UI mode support are not part of the supported interface.
- `CHANGELOG.md` now has a v0.20.3 checkpoint entry that describes the review as UI2-only and documents that runtime behavior and assets were preserved.
- Historical docs still mention UI1 where they describe prior audits, retirement decisions, or validation passes. Those references are intentionally historical and do not imply current support.

Remaining `.quickdeck-*` references were reviewed conservatively.

Safe/shared/core/runtime categories:

- `.quickdeck-open-button` launcher styling.
- `.quickdeck-restore-*` and `.gurps-quickdeck-minimized` minimized/restore-pill styling.
- `.quickdeck-reference-*`, `.quickdeck-pdf-*`, `.gurps-quickdeck-reference`, and `.gurps-quickdeck-reference-index` reference and local override windows.
- Runtime constants/selectors in `scripts/quickdeck-app.js` for the Foundry host, restore pill, reference apps, and overlay lifecycle plumbing.
- Historical docs that mention `.quickdeck-*` selector classification from earlier UI1 retirement and UI2 decoupling reports.

These remaining names are legacy labels, but the current uses are shared/core/runtime surfaces that are still part of the UI2-only module. Renaming them would be cosmetic refactor risk and is deferred to a future polish/refactor branch only if desired.
