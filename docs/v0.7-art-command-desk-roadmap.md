# GURPS QuickDeck v0.7.0 — Art-Filled Command Desk UI Roadmap

## Locked target

Use `design/reference/command-desk-target.png` as the visual target for the v0.7 art-filled UI.

The goal is to move QuickDeck from the cleaner v0.6.x CSS shell into an art-rich dark leather / bronze / parchment Command Desk interface without breaking any GURPS Game Aid behavior.

## Recommended success path

Do **not** build this as one giant background image.

Build the UI from reusable WebP pieces:

- dark leather tiles and panels
- brass rails and corners
- parchment drawer background and rows
- brass button states
- tab button states
- circular defense medallions
- footer/nameplate pieces
- small source-cropped icons only where useful

All shipped art uses `.webp` at quality `75`.

## Phase 1 — Asset foundation

Add this folder structure:

```text
assets/ui/command-desk/
  buttons/
  defense/
  dividers/
  footer/
  frames/
  icons/
  panels/
  parchment/
  reference/
  tabs/
```

Keep `design/reference/command-desk-target.png` in the repo only if you want source/reference material tracked. Otherwise keep it local and ship only the compressed WebP assets.

## Phase 2 — CSS-only skin file

Create:

```text
styles/quickdeck-command-desk.css
```

Add it to `module.json` after the existing QuickDeck stylesheet:

```json
"styles": [
  "styles/quickdeck.css",
  "styles/quickdeck-command-desk.css"
]
```

This keeps the art pass isolated.

## Phase 3 — Wire art without behavior changes

Rules:

- Do not change attack, defense, spell, skill, or actor logic.
- Do not touch `actor.sheet`.
- Do not remove or rename `data-action` hooks.
- Do not block clicks with decorative layers.
- Decorative pseudo-elements must use `pointer-events: none`.
- Preserve minimize / restore pill behavior.
- Preserve current Application v1 style.

## Phase 4 — Layout pass

Match the screenshot in these zones:

```text
+------------------------------------------------------+
| Command Desk header / title / sources / minimize     |
+------------------+--------------------+--------------+
| Left roster      | Selected actor      | Right drawer |
|                  | HP FP Move Defense  | Tabs + list  |
|                  | Favorites sections  |              |
+------------------+--------------------+--------------+
| Bottom GURPS footer / brass frame                    |
+------------------------------------------------------+
```

Target first around the screenshot aspect ratio, then add responsive clamps for smaller windows.

## Phase 5 — Interactive states

Add hover/active states for:

- top buttons
- red remove buttons
- green added buttons
- drawer tabs
- spell/skill action buttons
- defense medallions

## Phase 6 — Test checklist

1. Launch Foundry.
2. Open QuickDeck.
3. Confirm no missing asset paths in console.
4. Confirm roster add/select works.
5. Confirm actor portrait and selected actor data still work.
6. Confirm HP/FP controls still work.
7. Confirm Dodge / Parry / Block still roll.
8. Confirm Combat / Skills / Quick Skills / Spells / Settings tabs still switch.
9. Confirm drawer scrolls internally.
10. Confirm Sources and Minimize buttons still work.
11. Confirm restore pill left-click restore and right-drag move still work.
12. Confirm no decorative art blocks clicks.
13. Confirm no browser console errors.
14. Refresh Foundry and retest.
